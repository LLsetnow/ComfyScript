本文讨论了激活飞书长连接的相关内容，包括快速开始的步骤、项目结构、配置说明、事件处理、注意事项、常见问题解答等。关键要点包括：​
前置条件：需要 Node.js 16.0 或更高版本，npm 或 yarn 包管理器，飞书开发者账号和企业自建应用。​
快速开始步骤：克隆项目，安装依赖，配置环境变量（设置飞书应用的 App ID 和 App Secret，可使用 .env 文件管理），启动长连接客户端（npm run ws 或 npm start），配置飞书开发者后台（选择企业自建应用，进入事件与回调配置页面，选择使用长连接接收事件并订阅所需事件）。​
项目结构：包含 src/ws-client.js（长连接客户端主程序）、package.json（项目配置和依赖）等文件。​
事件处理：默认处理 im.message.receive_v1 事件，可在 src/ws-client.js 中添加更多事件处理器，常见事件类型有接收消息、消息已读、用户进群等。​
注意事项：仅支持企业自建应用，收到消息需在 3 秒内处理完成，每个应用最多建立 50 个连接，多客户端部署时只有随机一个客户端会收到消息。​
常见问题解答：启动提示缺少环境变量需确保正确设置；开发者后台保存订阅方式失败需确保本地客户端启动且连接成功；收不到事件消息需检查订阅事件、应用权限和长连接状态；生产环境部署可使用 PM2 进行进程管理。​

Q: 开发者后台保存订阅方式失败？​
​
A: 必须确保本地客户端已启动且连接成功后，才能在后台保存「使用长连接接收事件」的配置。​
​
Q: 为什么收不到事件消息？​
​
A: 请检查：​
是否在开发者后台正确订阅了对应事件​
应用是否有对应的权限​
长连接是否正常建立（查看控制台日志）​
Q: 如何在生产环境部署？​
​
A: 可以使用 PM2 进行进程管理：​
​
